# Quiniela Mundial 2026 

Static fantasy-scoring site for the 2026 World Cup group stage.
Deployed on **Render** (auto-deploys on push to `main`).
→ https://quiniela-7nl4.onrender.com

## Adding a participant

1. Drop their `name.xlsx` into `data/players/`
2. Add their display name to `data/nicknames.json`:
   ```json
   "name.xlsx": "Nickname goes here"
   ```
   Nickname priority: `nicknames.json` → `Home!C10` in Excel → filename
3. `git add -A && git commit -m "add: nickname"` — the pre-commit hook runs
   `prerender.py` automatically and stages a fresh `scores.json`
4. `git push` — Render deploys in ~1 min

> **Spaces in filenames are fine** — `encodeURIComponent` handles them in `app.js`.

### Re-installing the pre-commit hook (after a fresh clone)

```bash
cp hooks/pre-commit .git/hooks/pre-commit && chmod +x .git/hooks/pre-commit
```

## Committing without running prerender

The pre-commit hook runs `prerender.py` on **every** commit, which:
- Re-scores all players
- Writes a fresh `scores.json`
- **Stores current positions as `prevPosition`** for the next run's up/down tracking

For **UI-only changes** (JS, HTML, CSS) where you don't want to touch scores or
freeze positions, skip the hook entirely:

```bash
git add js/ui.js               # or whatever files changed
git commit --no-verify -m "ui: your message"
git push
```

`--no-verify` skips all pre-commit hooks. Use it any time you're not changing
`master.xlsx` or player files. You'll still need to run `python3 prerender.py`
manually if you want a local scores refresh.

> **Rule of thumb:** only do a normal `git commit` (with hook) when you have
> updated real match results in `master.xlsx` or added/changed a player file.
> Everything else gets `--no-verify`.

## Updating real results

1. Open `data/master.xlsx` → fill **AC** (home goals) and **AD** (away goals) for
   played matches in the `WORLDCUP` sheet
2. **Important:** use `Delete` to clear unplayed cells — typing `0` counts as a
   real 0-0 result, not "not played"
3. Save → `git add -A && git commit -m "results: ..."` → `git push`

## Scoring rules

| Situation | Points |
|-----------|--------|
| Exact score (both goals correct) | 6 |
| Correct winner/draw + one goal count correct | 4 |
| Correct winner/draw, no goal count correct | 3 |
| Wrong outcome but one goal count correct | 1 |
| Nothing correct | 0 |
| Match not yet played | — (null) |

**Standings bonus:** `BONUS_PER_POSITION = 1` pt per correctly predicted final group
position, applied only when all 6 group matches are played.
Constant lives in **`js/scorer.js`** and is replicated in **`prerender.py`** — keep in sync.

## UI features

- **Sidebar (desktop):** fixed left panel, player list sorted by total pts
- **Mobile:** sticky top bar with dropdown select; match tables show FIFA 3-letter
  country codes (`GER`, `ARG`, `BRA`…) instead of full names to avoid horizontal scroll
- **Group standings table:** always visible below each group's matches, showing
  predicted order with columns `Pts G E P GF GC DG`
  - While group is incomplete: shows predicted positions + `(bono al terminar grupo — X/6)`
  - When all 6 matches are played: adds `Real` vs `Pred.` comparison and `Bono` column

## Excel layout (WORLDCUP sheet)

Groups A–L, indexed `i = 0..11`:

| Data | Rows | Column |
|------|------|--------|
| Match fixtures | `4+8i` … `9+8i` | AA=home team, AC=home goals, AD=away goals, AF=away team |
| Team names | `5+8i` … `8+8i` | A |
| Standings | `6+4i` … `9+4i` | BD |

Player name: `Home!C10` (falls back to filename if blank or `"Nombre"`).

>  To add a new participant their file **must** have a sheet named `WORLDCUP`.
> If the sheet is missing, `prerender.py` skips the file with a warning.
