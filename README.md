## How the pre-commit hook works (and why it doesn't loop)

The hook lives at `hooks/pre-commit` (copy to `.git/hooks/pre-commit` after a fresh clone).

**What happens on `git commit`:**

```
git commit -m "results: ..."
    │
    ├─ Git invokes .git/hooks/pre-commit (ONCE, before writing the commit)
    │       │
    │       ├─ Checks: are any .xlsx files staged?
    │       │     No  → exits immediately, commit proceeds unchanged
    │       │     Yes → runs python3 prerender.py
    │       │               └─ writes updated scores.json to disk
    │       └─ runs git add data/scores.json  ← stages it into the same commit
    │
    └─ Git finalises the commit with everything staged (including scores.json)
       DONE — the hook does not call git commit, so there is no second trigger
```

**Why no infinite loop:** A pre-commit hook runs *before* the commit is written.
It can modify and stage files, then hands control back to git which writes
exactly one commit. `git add` inside a hook does not trigger another hook.
The hook never calls `git commit`, so there is nothing to loop.

**Why idempotent runs don't corrupt history:** `prerender.py` compares the newly
computed positions against the last entry in each player's `positionHistory`.
If nothing changed (e.g. the hook ran, then `serve.sh`'s watcher also ran, then
the hook ran again), the array simply isn't appended to. Run it 100 times with
the same `master.xlsx` — `positionHistory` grows by exactly one entry.

## Committing — the smart hook

The pre-commit hook **only runs prerender when an `.xlsx` file is staged**:

| What you staged | Hook behaviour |
|----------------|----------------|
| `master.xlsx` or any player `.xlsx` | Runs prerender, updates `scores.json`, saves positions |
| JS / HTML / CSS / docs only | Skips prerender — positions stay untouched |

```bash
# With xlsx changes (new result, new player):
git add -A && git commit -m "results: ..." && git push

# UI-only changes (JS/HTML/CSS):
git add js/ui.js && git commit --no-verify -m "ui: ..." && git push
```

> **For Code Puppy:** always check `git diff --cached --name-only` before deciding
> whether to run prerender. If no `.xlsx` is staged, use `--no-verify`.
> Never run prerender manually just before a normal commit — it will cause
> double-tracking of positions.

### Re-installing the hook after a fresh clone

```bash
cp hooks/pre-commit .git/hooks/pre-commit && chmod +x .git/hooks/pre-commit
```

## Adding a participant

1. Drop their `name.xlsx` into `data/players/`
2. Add nickname to `data/nicknames.json`:
   ```json
   "name.xlsx": "Nickname \"alias\" Surname"
   ```
3. Add bio to `data/descriptions.json`:
   ```json
   "name.xlsx": "City, age.\nLine two.\nLine three."
   ```
   Use `\n` for line breaks — rendered with `whitespace-pre-line`.
4. Optionally add photo — see below
5. `git add -A && git commit -m "add: nickname"` — hook runs prerender automatically
6. `git push` — Render deploys in ~1 min

## Adding / updating profile photos

Photos live in `data/photos/`. Any filename, any extension (jpg/jpeg/png/webp).

1. Drop the raw photo into `data/photos/`
2. Add mapping to `data/photos.json`:
   ```json
   "PlayerFile.xlsx": "photoname.jpeg"
   ```
3. Run the smart crop script (face detection, 1:1 square output):
   ```bash
   /opt/homebrew/Cellar/python@3.13/3.13.3/Frameworks/Python.framework/Versions/3.13/bin/python3.13 crop_photos.py
   ```
   Falls back to center crop if no face detected. Overwrites files in place.
4. `git add data/photos/ data/photos.json && git commit --no-verify -m "photos: ..."`

## Updating real results

1. Open `data/master.xlsx` -> fill **AC** (home goals) and **AD** (away goals)
2. **Important:** `Delete` key = unplayed (null). Typing `0` = a real 0-0 result.
3. `git add data/master.xlsx && git commit -m "results: ..." && git push`

## Scoring rules

| Situation | Points |
|-----------|--------|
| Exact score | 6 |
| Correct winner/draw + one goal correct | 4 |
| Correct winner/draw, no goal correct | 3 |
| Wrong outcome but one goal correct | 1 |
| Nothing correct | 0 |
| Match not yet played | — (null) |

**Standings bonus:** `BONUS_PER_POSITION = 1` pt per correctly predicted final group
position, applied only when all 6 group matches are played.
Constant lives in **`js/scorer.js`** and **`prerender.py`** — keep in sync.

**Tiebreaker order:** Pts -> 6P -> 4P -> 3P -> 1P -> alphabetical

## scores.json structure

```json
{
  "players": [
    {
      "file": "david.xlsx",
      "displayName": "David \"el chino\" Caicedo",
      "totalPoints": 12,
      "counts": { "p6": 1, "p4": 1, "p3": 0, "p1": 0, "p0": 0 },
      "position": 1,
      "prevPosition": 2
    }
  ]
}
```

`prevPosition` is only set if the previous run had at least one scored player
(avoids storing the meaningless all-zero pre-game ordering).

## Excel layout (WORLDCUP sheet)

Groups A-L, indexed `i = 0..11`:

| Data | Rows | Column |
|------|------|--------|
| Match fixtures | `4+8i` ... `9+8i` | AA=home, AC=home goals, AD=away goals, AF=away |
| Team names | `5+8i` ... `8+8i` | A |
| Standings | `6+4i` ... `9+4i` | BD |
